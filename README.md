## 1. КОНЦЕПТУАЛЬНИЙ ТЕОРЕТИЧНИЙ БАЗИС

### 1.1 Мета та об'єкт дослідження
Метою даної науково-практичної роботи є вивчення фундаментальних принципів взаємодії користувача з ядром операційної системи через командний інтерпретатор **Bash (Bourne Again SHell)**. В ході роботи досліджується логіка обробки текстових потоків, механізми управління змінними оточення, створення користувацьких макросів (аліасів) та розробка скриптових модулів для автоматизації рутинних адміністративних задач.

### 1.2 Глосарій розширеної термінології

Нижче наведено структурований словник базових дефініцій, що використовуються в архітектурі Unix-подібних систем:

| Технічний термін | Семантичне значення | Функціональна роль у системі |
|:-----------------|:--------------------|:-----------------------------|
| **Command** | Системний виклик | Атомарна інструкція, що ініціює виконання двійкового коду або вбудованої функції. |
| **CLI (Interface)** | Текстова оболонка | Метод керування ресурсами без залучення графічного стека (X11/Wayland). |
| **Kernel Shell** | Інтерпретатор | Програмний прошарок, що виступає парсером команд між користувачем та ядром. |
| **Prompt (PS1)** | Маркер готовності | Рядок динамічного запиту, що відображає метадані поточної сесії (User, Host, PWD). |
| **Option/Flag** | Модифікатор логіки | Ключі (Short/Long options), що змінюють параметри виконання утиліти. |
| **Alias** | Псевдонім | Метод абстракції складних командних ланцюгів у короткі ідентифікатори. |
| **Man Pages** | Технічний репозиторій | Локальна база документації, що містить специфікації до кожної системної утиліти. |
| **Directory/Path** | Ієрархія ФС | Адресація об'єктів у деревоподібній структурі файлової системи Linux. |

## 2. АНАЛІТИЧНИЙ РОЗБІР КОМАНДНОГО СТЕКА

### 2.1 Ієрархія та типи команд у Bash
В системі Linux виділяють чотири основні класи команд, кожна з яких має свій пріоритет виконання:
1.  **Aliases:** Користувацькі скорочення (найвищий пріоритет).
2.  **Functions:** Локальні процедури, оголошені в сесії або `.bashrc`.
3.  **Built-ins:** Команди, вбудовані безпосередньо в код оболонки (напр., `cd`, `pwd`, `echo`).
4.  **External Programs:** Бінарні файли, розташовані в директоріях, вказаних у змінній `$PATH` (напр., `/usr/bin/ls`).

### 2.2 Управління параметрами та аргументами
Кожна команда в CLI-режимі підпорядковується стандарту POSIX:
* **Опції** (наприклад, `ls -la`) — змінюють "як" працює команда.
* **Аргументи** (наприклад, `ls /home/user`) — вказують "над чим" виконується дія.

## 🛠️ 3. ПРАКТИЧНИЙ ПРОТОКОЛ ТА ХІД РОБОТИ

### 3.1 Моделювання середовища та робота зі змінними
Змінні в Bash є фундаментальним інструментом для передачі даних між процесами. Ми ініціалізували три строкові змінні для імітації бази даних користувачів.

```bash
# --- Етап 1: Ініціалізація локальних змінних ---
# Створюємо змінні для ідентифікації учасників групи
user_node_01="Dmytro"
user_node_02="Denys"
user_node_03="Oleksandr"

# --- Етап 2: Верифікація даних через стандартний вивід ---
# Використовуємо команду echo з інтерполяцією змінних
echo "Ініціалізовано записи для вузлів: $user_node_01, $user_node_02, $user_node_03"

# --- Етап 3: Аудит системних змінних оточення ---
# Перегляд поточної директорії та імені хоста
echo "Поточна локація сесії: $PWD"
echo "Активний користувач у системі: $USER"

3.  **Змінна `$PS1`:** Визначає структуру рядка запиту. Вона дозволяє динамічно відображати поточний час, шлях або статус попередньої команди.
